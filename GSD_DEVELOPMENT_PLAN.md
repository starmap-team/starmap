## 五、开发规范

### 5.1 Git 工作流

```
分支命名: r{角色编号}/{功能描述}
Commit格式: {type}(R{n}): {描述}
PR流程: feature分支 → push → gh pr create → CI全绿 → squash merge
CI门禁: 契约校验 → 后端(ruff+mypy+pytest) → 前端(lint+typecheck+build) → 爬虫
```

### 5.2 代码质量标准

| 标准 | 门禁值 | 当前值 |
|------|--------|--------|
| 后端测试覆盖率 | ≥60% | 65.43% |
| Ruff lint | 0 errors | 0 |
| Mypy | 0 errors | 0 |
| TypeScript | 0 errors | 0 |
| ESLint | ≤25 warnings | 18 |
| CI | 4/4 pass | ✅ |

### 5.3 分支保护规则

- main分支必须通过4个CI check
- PR分支必须基于最新main(strict mode)
- 合并后自动删除feature分支
- squash merge保持commit历史干净
