## 2. 标注 Schema

每条 JD 标注为 JSONL(一行一 JSON):

```json
{
  "id": "jd-001",
  "job_title": "高级前端开发工程师",
  "required_skills": ["React", "TypeScript", "JavaScript", "HTML5", "CSS3", "Webpack", "Git"],
  "bonus_skills": ["Next.js", "Tailwind CSS", "GraphQL", "Node.js"],
  "experience_years": 5,
  "education": "本科及以上"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识,jd-NNN |
| job_title | string | 标准化岗位名(去公司前缀、去级别修饰归一) |
| required_skills | string[] | **必备技能**(JD 明确要求的) |
| bonus_skills | string[] | 加分项技能(JD 写"优先""加分""了解") |
| experience_years | int | 要求年限(取区间下限,如"3-5年"记3) |
| education | string | 学历门槛,枚举值见下 |

## 3. 核心标注原则(评审沉淀)

### 3.1 技能颗粒度必须一致 — 严禁粗颗粒度词

**规则:只标具体技能(工具/框架/语言/协议/方法),不标领域或职责描述。**

| ❌ 禁用(粗颗粒度) | ✅ 应标为(具体技能) |
|-------------------|---------------------|
| 机器学习 | scikit-learn, XGBoost, LightGBM |
| 深度学习 | PyTorch, TensorFlow, Keras |
| 数据分析 | SQL, Pandas, Excel, Tableau |
| 项目管理 | Jira, Confluence, PMP, Scrum |
| 前端开发 | React, Vue, TypeScript |

**例外**:产品经理、项目经理等非技术岗位的软技能(需求分析、沟通协调)可保留,但需单列说明。

### 3.2 required vs bonus 的判定

| 判定 | 依据 |
|------|------|
| required | JD 用"精通""熟练掌握""必须""要求" |
| bonus | JD 用"优先""加分""了解""熟悉即可" |
| 模糊 | 默认归 required(从严,避免漏判) |

### 3.3 技能归一化(评估时)

标注阶段保留原始技能名,归一化在评估时进行(见 §14.3.1):
- 别名映射:`Python3` → `Python`、`ML` → `机器学习`(由 normalize.py 处理)
- 向量相似度 ≥0.85 视为同一技能
- **标注时不预先归一化**,保留多样性以测归一化能力

## 4. 数据分布要求

为避免样本偏差导致 F1 失真,标注需覆盖:

### 4.1 岗位覆盖(50条分布)
- 前端/后端/全栈: 8条
- AI/算法: 12条(含 CV/NLP/推荐/强化学习/大模型)
- 数据: 8条(分析/工程/科学/仓库/治理/BI)
- 运维/DevOps/云: 5条
- 移动/游戏/嵌入式: 6条
- 测试/安全: 5条
- 管理/其他: 6条

### 4.2 experience_years 分布(避免均一)
- 应届/初级(0-2年): ~10条
- 中级(3-5年): ~25条
- 资深/专家(6+年): ~15条

### 4.3 education 分布
- 本科及以上(主体)
- 硕士及以上(算法岗为主)
- 大专及以上 / 不限 / 博士(各少量,保证真实性)

### 4.4 required_skills 数量波动
- 真实 JD 技能数差异大,标注应覆盖 3-9 个,避免全部 6-7 个的模板化

## 5. 质量门禁(评估时启用)

```
JD解析准确率 F1 加权:
  required_skills F1 × 0.30
  bonus_skills F1 × 0.20
  job_title × 0.15
  experience_years(误差≤1年正确) × 0.15
  education(精确匹配) × 0.20

门禁:
  F1 ≥ 0.90 绿色(M7目标)
  F1 ≥ 0.85 黄色(M4中期)
  F1 ≥ 0.80 橙色(预警)
  F1 < 0.80 红色(技术负责人介入)
```
